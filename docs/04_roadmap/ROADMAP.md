# Roadmap

This roadmap tracks product work that is not yet part of the durable gameplay,
technical or UI documentation. When a feature becomes implemented and stable,
move its durable rules to the relevant document and remove it from this file.

## Near-Term Priorities

Goal: make weekly decisions richer without adding repetitive micromanagement.

- Add clearer readiness explanations, including a visible factor breakdown for health, energy, morale, satiety and reputation stability.
- Improve alert prioritization so the most urgent blockers and risks surface before lower-value warnings.
- Deepen weekly recommendations beyond simple gauge thresholds and objective-to-building mapping.
- Balance building policies, improvements, upgrade costs, Domus capacity pacing and building effects.
- Improve contract variety, contract reward tuning and contract risk readability.
- Improve event variety and event consequence readability while keeping event frequency understandable.
- Complete the betting loop with actual bet placement, resolution, payouts and loss handling.
- Expand scouting from basic opponent reveal into deeper opponent information and risk/reward decisions.

## Content Backlog

Goal: expand variety while keeping systems understandable.

- More gladiator traits.
- More names and visual identities.
- More market candidate variety.
- More weekly contract definitions.
- More daily event definitions.
- Additional building improvements.
- Additional building policies.
- Richer combat log text.
- More arena rank flavor and rank-specific rewards or risks.
- Richer map state feedback and optional flag/banner variations.
- More demo scenarios when needed for manual review or smoke coverage.

Content should use i18n keys and remain data-driven wherever possible.

## Advanced Systems

Goal: add strategic depth only when it reinforces the weekly preparation and
Sunday resolution loop.

- Public perception manipulation.
- Severe injury rules.
- Gladiator death rules.
- Staff or employee system.
- Rival ludi.
- Richer event chains.
- Real cloud save backend.
- Authentication.
- More advanced map movement or pathfinding.

## Polish And Production

Goal: make the game readable, comfortable and emotionally sticky.

- Improve empty, warning and error states where current panels are still thin.
- Improve accessibility and keyboard/readability coverage.
- Improve performance for larger rosters.
- Reintroduce high-value Playwright smoke coverage for stable map, save and arena flows.
- Replace temporary renderer placeholder art with authored spritesheets and building assets.
- Add audio if it supports the atmosphere without hurting usability.

## Open Product Decisions

- Exact long-term building level count.
- Exact final building upgrade costs and future optional building purchase costs.
- Exact improvement costs.
- Exact food and entertainment formulas.
- Final combat probability formulas.
- Final betting rules.
- Exact contract reward formulas.
- Long-term event frequency and event-chain depth.
- Exact cloud save backend.
- Exact authentication system.
- Gladiator death and severe injury rules.
- Employee/staff system.
- Final asset production workflow.
